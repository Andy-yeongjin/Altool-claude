Altool 패치노트

원본 Altool/을 기준으로, 업데이트 배포판인 Altool-codex/와 Altool-claude/에서 무엇이 바뀌었는지 기존 사용자가 보기 쉽게 정리했습니다.

기준 원본: Altool/ 패치 대상: Altool-codex/, Altool-claude/ 비교일: 2026-07-09 비교 방식: 현재 폴더 스냅샷 기준 제외: .git, 실행 상태 파일 .altool

한눈에 보는 핵심 변경

이번 패치는 단순 문구 수정이 아니라, 원본 Altool을 Codex와 Claude Code 양쪽에서 바로 쓸 수 있는 배포판으로 나눈 업데이트입니다.

명령어 체계 변경

원본의 /al 중심 흐름이 Codex판 $altool, Claude판 /altool로 분리되었습니다.

Research-first 도입

개발 전에 웹 조사와 디자인 시스템 정규화를 먼저 수행하도록 research 단계가 추가되었습니다.

Claude 디자인 HTML 우선

designs/claude-design/*.html을 최우선 디자인 헌법으로 보고, oneshot과 freedom에서 자동 정규화합니다.

자율 반복 개발 추가

freedom 모드가 추가되어 지정한 횟수만큼 조사, 구현, 검증, 보고 사이클을 반복할 수 있습니다.

실제 브라우저 검증 강화

browser 단계가 정식으로 추가되어 클릭, 입력, 반응형, 콘솔 오류, 화면 깨짐까지 확인합니다.

Step Check 검증 도구 추가

각 단계 완료 시 체크 JSON을 만들고 check.py로 필수 증거 누락을 검증합니다.

내장 성능 가이드 추가

React/Next.js 작업에 적용할 Vercel React Best Practices 스킬과 규칙 문서가 번들로 들어갔습니다.

원본 대비 변경 규모

Git 이력 비교가 아니라 현재 폴더 스냅샷 기준으로 배포 관점 파일만 비교했습니다. 실행 중 생성되는 .altool/ 상태 파일과 저장소 메타데이터인 .git/은 제외했습니다.

Altool-codex +95 / ~27 / -8 추가 95개, 수정 27개, 제거 8개
Altool-claude +93 / ~27 / -8 추가 93개, 수정 27개, 제거 8개

두 패치판은 엔진 개선 내용이 거의 같고, 설치 대상에 맞춰 Codex용 로컬 스킬 구조와 Claude Code용 로컬 커맨드 구조만 다릅니다.

기존 사용자에게 중요한 변화

영역 원본 Altool 패치 후 사용자 영향

Claude Code: /altool		 기존 명령어를 새 배포판에 맞게 바꿔 입력해야 합니다.

디자인 자동화 디자인 소스는 별도 단계로 실행 oneshot/freedom 시작 시 design_source 자동 실행 HTML을 designs/claude-design/에 넣고 바로 개발 명령을 실행해도 디자인 시스템이 먼저 갱신됩니다.
검증 방식 문서 체크와 Match Rate 중심 Step Check JSON, 브라우저 검증, CSS 변수 검사, 문서 상태 동기화 완료 조건이 더 빡빡해져 누락된 산출물이나 검증 증거를 잡기 쉬워졌습니다.
장기 작업 단일 명령 중심 freedom, say, ask, pause, resume, stop 긴 작업 중간에 방향 변경, 질문, 일시정지, 중단을 파일 기반 무전기 방식으로 처리합니다.

새 명령어와 파이프라인

패치판의 oneshot은 원본보다 앞단 조사와 뒷단 브라우저 검증이 강화된 7단계 흐름입니다.

1. research 2. plan 3. spec 4. run 5. analyze 6. fix 7. browser
새 명령 설명
research 실제 서비스, UX 자료, 사용자 요구사항, 리스크 자료를 조사하고 docs/00-research/에 기록합니다.
spec 기존 design 역할을 확장해 아키텍처, API, 데이터 모델, UI 체크리스트, 테스트 계획을 명세화합니다.
browser 실제 브라우저 또는 Playwright로 주요 화면, 클릭, 입력, 반응형, 콘솔 오류, CSS 변수 누락을 확인합니다.
freedom 지정한 루프 수만큼 research부터 report까지 자율 개발 사이클을 반복합니다.
say / ask / pause / resume / stop / outbox Freedom 루프 진행 중 방향 전달, 상태 질문, 일시정지, 재개, 중단, 응답 확인을 처리합니다.

패치판별 차이

둘 다 원본 Altool을 개선한 버전이며, 사용 중인 AI 도구에 맞춰 선택하면 됩니다.

Codex 버전

Altool-codex

  • 명령어: $altool
  • 진입 문서: AGENTS.md
  • 로컬 스킬: .agents/skills/altool/
  • 설치 시 templates/codex/skills/를 프로젝트에 복사
  • Codex 환경에서 바로 쓰기 위한 openai.yaml 포함
Claude Code 버전

Altool-claude

  • 명령어: /altool
  • 진입 문서: CLAUDE.md
  • 로컬 커맨드: .claude/commands/altool.md
  • 설치 시 templates/claude/commands/와 templates/claude/skills/를 프로젝트에 복사
  • Claude Code 재시작 또는 새 세션 후 명령 사용

설치와 업그레이드 메모

기존 프로젝트에 적용할 때는 새 배포판의 setup.bat을 실행해 대상 프로젝트 폴더를 선택하면 됩니다. 기존 .gitignore는 덮어쓰지 않고, 필요한 경우 .altool/ 제외 규칙을 직접 추가하라는 안내만 표시합니다.
항목 변경 내용
setup.bat 비대화형 경로 인자 지원, 폴더 선택 실패 시 수동 입력, Codex/Claude별 로컬 스킬 또는 커맨드 복사, designs/claude-design/ 폴더 생성 추가.
LICENSE, THIRD_PARTY_NOTICES.md MIT 라이선스와 번들된 제3자 자료 고지를 명시.
README.md 명령어, 설치 방법, Research-first, Freedom, 브라우저 검증, 패치판별 사용법을 새로 정리.
Vercel React Best Practices React/Next.js 성능, 렌더링, 서버, 번들, 클라이언트, 비동기 처리 관련 규칙을 로컬 스킬로 번들.

제거되거나 통합된 항목

삭제는 기능 축소라기보다 새 명령 체계와 단일 디자인 기준으로 정리된 결과입니다.

  • altool/steps/design.md와 altool/templates/design.template.md는 spec 단계로 대체되었습니다.
  • designs/design-constitution.md와 designs/design-tokens.css는 designs/design.md 중심 정책으로 통합되었습니다.
  • 원본의 templates/.claude/commands/al.md와 templates/CLAUDE.md는 패치판별 구조로 이동했습니다.
  • 원본 내부의 altool/CLAUDE.md는 루트 AGENTS.md 또는 CLAUDE.md와 로컬 커맨드/스킬 문서로 재배치되었습니다.
